## 3. 주요 발견사항 (Top 7)

### 1) [심각: 긴급] WS-31 (SSL 검증 비활성화)
- **파일**: `app/main.py` (L40)
- **위험**: 외부 API 호출 시 SSL 인증서 검증을 수행하지 않음 (`verify=False`). Man-in-the-Middle (MitM) 공격을 통한 민감 데이터 탈취 위험.
- **권고**: `verify=True`로 변경하고 신뢰할 수 있는 CA 인증서를 사용하십시오.

### 2) [심각: 높음] WS-36 (민감 정보 평문 저장)
- **파일**: `ui/src/stores/collections.ts` (L178, L182)
- **위험**: `localStorage`를 사용하여 인증 토큰 및 패스워드가 포함된 컬렉션 데이터를 브라우저에 평문으로 저장.
- **권고**: 중요 정보(Secrets)는 암호화하여 저장하거나, 메모리(Store) 내에서만 유지하고 필요 시 세션 스토리지 사용을 검토하십시오.

### 3) [심각: 높음] WS-13 (부적절한 CORS 설정)
- **파일**: `app/main.py` (L15)
- **위험**: `allow_origins=["*"]` 설정으로 모든 도메인에서의 접근을 허용. 악성 사이트에서 사용자의 API 세션을 이용한 요청 가능.
- **권고**: 실제 서비스되는 도메인으로 `allow_origins`를 제한하십시오.

### 4) [심각: 높음] WS-41 (SSRF 취약점 - 서버 측 요청 위조)
- **파일**: `app/main.py` (L40)
- **위험**: 백엔드(`/execute`)가 검증되지 않은 외부 URL로 요청을 대행. 내부망 스캐닝 또는 클라우드 메타데이터(169.254.169.254) 탈취에 악용 가능.
- **권고**: 허용된 도메인 리스트(Whitelist)를 운영하거나, 내부 IP 대역(RFC 1918)으로의 요청을 차단하는 로직을 추가하십시오.

### 5) [심각: 높음] WS-01 (XSS - Iframe 격리 미흡)
- **파일**: `ui/src/components/ResponseViewer.vue` (L183)
- **위험**: 응답 HTML을 렌더링하는 `iframe`에 `sandbox` 속성이 없어, 렌더링된 악성 HTML이 부모 창의 `localStorage`나 쿠키에 접근 가능.
- **권고**: `<iframe sandbox="allow-scripts allow-forms" ...>`와 같이 `sandbox` 속성을 추가하여 권한을 제한하십시오.

### 6) [심각: 높음] U-46 (로그 관리 부재)
- **파일**: 전체 시스템
- **위험**: API 실행 내역, 에러 로그, 관리자 접근 로그가 기록되지 않음. 침해 사고 발생 시 추적 및 대응 불가능.
- **권고**: Python `logging` 모듈을 사용하여 파일 또는 외부 로그 서버로 주요 이벤트(로그인, 실행, 에러)를 기록하십시오.

### 7) [심각: 보통] WS-01 (Command Injection 잠재 위험)
- **파일**: `app/main.py` (L101)
- **위험**: `subprocess.run(..., shell=True)` 사용. 현재는 하드코딩되어 있으나 파라미터화될 경우 명령 실행 취약점 발생 가능.
- **권고**: `shell=False`를 사용하고 리스트 형태의 인자를 전달하십시오.
