# KESE 주요정보통신기반시설 취약점 분석평가

## 1단계: 환경 탐지

### 자동 탐지
1. **운영체제**: 파일 패턴 및 설정 파일로 탐지
   - Unix/Linux: `/etc/passwd`, `/etc/shadow`, 쉘 설정
   - Windows: 레지스트리, PowerShell, IIS 설정
2. **웹 서비스**: 웹 서버 유형 (Apache, Nginx, IIS, Tomcat)
3. **데이터베이스**: DB 유형 (Oracle, MySQL, PostgreSQL, MSSQL 등)
4. **네트워크 장비**: 라우터/스위치/방화벽 설정
5. **클라우드 환경**: AWS/Azure/GCP 탐지

## 2단계: 기술적 취약점 분석 (424항목)

### Unix/Linux 취약점 분석 (U-01 ~ U-68)

| 분류 | 항목 | 주요 점검 내용 |
|------|:----:|---------------|
| 계정관리 | U-01~U-09 | root 원격 접속, 패스워드 정책, 계정 잠금 |
| 파일/디렉터리 | U-10~U-24 | 권한 설정, SUID/SGID, umask |
| 서비스 관리 | U-25~U-43 | 불필요 서비스, 보안 설정 |
| 패치 관리 | U-44~U-45 | 시스템 패치, 보안 업데이트 |
| 로그 관리 | U-46~U-49 | 감사 로그, 로그 보존 |

**주요 취약점 점검 명령어:**
```bash
# U-01: root 원격 접속 제한
grep "^PermitRootLogin" /etc/ssh/sshd_config

# U-02: 패스워드 복잡도
grep "minlen" /etc/security/pwquality.conf

# U-05: Shadow 파일 권한
ls -la /etc/shadow

# U-10: SUID/SGID 파일
find / -perm -4000 -o -perm -2000 2>/dev/null
```

### Windows 취약점 분석 (W-01 ~ W-73)

| 분류 | 항목 | 주요 점검 내용 |
|------|:----:|---------------|
| 계정관리 | W-01~W-15 | Administrator 계정, 패스워드 정책 |
| 서비스 관리 | W-16~W-35 | 불필요 서비스, 공유 설정 |
| 패치 관리 | W-36~W-40 | Windows Update, 보안 패치 |
| 로그 관리 | W-41~W-48 | 이벤트 로그, 감사 정책 |
| 보안 관리 | W-49~W-73 | 레지스트리, 화면보호기, 백신 |

**주요 취약점 점검 명령어:**
```powershell
# W-01: Administrator 계정명 변경
Get-LocalUser | Where-Object {$_.SID -like "*-500"}

# W-02: Guest 계정 비활성화
Get-LocalUser -Name "Guest" | Select-Object Enabled

# W-15: 패스워드 정책
net accounts
```

### 웹 서비스 취약점 분석 (WS-01 ~ WS-47)

| 분류 | 항목 | 주요 점검 내용 |
|------|:----:|---------------|
| 입력값 검증 | WS-01~WS-10 | SQL Injection, XSS, Command Injection |
| 보안 기능 | WS-11~WS-25 | 인증, 세션, 접근 제어 |
| 에러 처리 | WS-26~WS-30 | 에러 메시지, 예외 처리 |
| 암호화 | WS-31~WS-40 | 암호화, 안전한 전송 |
| 설정 관리 | WS-41~WS-47 | 서버 하드닝, 디렉터리 리스팅 |

**주요 OWASP 취약점 패턴:**
- SQL Injection: 쿼리 문자열 연결
- XSS: 이스케이프 없는 출력, innerHTML
- CSRF: 토큰 누락
- 경로 조작: `../` 포함 경로
- 안전하지 않은 역직렬화: pickle, yaml.load()

## 우선순위

### 긴급 (즉시 조치 필요)
- 계정 관리 취약점 (기본 패스워드, 잠금 없음)
- 미패치된 중요 취약점
- 인젝션 취약점 (SQL, XSS, Command)
- 민감 데이터 미암호화
- 관리자 인터페이스 노출

### 높음 (일정 내 조치)
- 설정 취약점
- 보안 헤더 누락
- 불완전한 로깅
- 약한 패스워드 정책

- 하드닝 권고사항
- 문서화 부족
- 프로세스 개선
